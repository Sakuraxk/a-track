数据结构

列表 (List) / 字典 (Dictionary) / 元组 (Tuple) / 集合 (Set)
四大内置数据结构的全面解析与交互练习

为什么需要数据结构?

一个现实场景

假设你需要管理一个班级的学生信息:

如果用单独的变量来存储:

student1 = "张三" student2 = "李四" student3 = "王五" # 如果有100个学生呢? # 100个变量? 显然不合理!

这就是数据结构存在的意义 -- 用一个结构管理一组相关数据。

Python 四大内置数据结构

列表 List
有序, 可变, 可重复
字典 Dictionary
键值对, 可变, 键唯一
元组 Tuple
有序, 不可变, 可重复
集合 Set
无序, 可变, 元素唯一

列表 (List): 最常用的数据结构

列表的特征

有序: 元素按插入顺序排列, 有固定的索引位置
可变: 创建后可以添加、删除、修改元素
可重复: 允许包含重复的元素
异构: 可以混合存放不同类型的数据
# 创建列表 fruits = ["apple", "banana", "cherry"] mixed = [1, "hello", True, 3.14] empty = [] nested = [[1, 2], [3, 4]] # 嵌套列表 # 用 list() 构造 chars = list("hello") # ['h','e','l','l','o'] nums = list(range(5)) # [0,1,2,3,4]

列表索引与切片

索引从0开始, 支持负索引(从末尾倒数)

fruits = ["apple", "banana", "cherry"] fruits[0] # "apple" fruits[-1] # "cherry" fruits[0:2] # ["apple", "banana"] fruits[1:] # ["banana", "cherry"] fruits[:2] # ["apple", "banana"]

列表: 常用方法

修改列表

fruits = ["apple", "banana"] # 添加元素 fruits.append("cherry") # 末尾添加 fruits.insert(1, "grape") # 指定位置插入 fruits.extend(["mango"]) # 合并另一个列表 # 删除元素 fruits.remove("banana") # 按值删除 fruits.pop(0) # 按索引删除并返回 del fruits[0] # 按索引删除 fruits.clear() # 清空列表

查询与排序

nums = [3, 1, 4, 1, 5, 9] # 查询 len(nums) # 6 -- 长度 nums.count(1) # 2 -- 出现次数 nums.index(4) # 2 -- 第一次出现的位置 5 in nums # True -- 是否存在 # 排序 nums.sort() # 原地升序: [1,1,3,4,5,9] nums.sort(reverse=True) # 降序 nums.reverse() # 反转 new = sorted(nums) # 返回新列表, 不修改原列表

列表推导式: squares = [x**2 for x in range(10)]

条件过滤: evens = [x for x in nums if x % 2 == 0]

列表: 交互式操作台

列表可视化

点击按钮操作列表, 观察变化

range(10) → 过滤偶数 → 平方

字典 (Dictionary): 键值对存储

字典的特征

键值对: 每个元素由 key:value 组成
键唯一: 不允许重复的键, 值可以重复
可变: 可以添加、修改、删除键值对
键不可变: 键必须是不可变类型(str, int, tuple)
# 创建字典 student = { "name": "张三", "age": 20, "grades": [85, 92, 78] } # 其他创建方式 d1 = dict(name="李四", age=21) d2 = dict([("a",1), ("b",2)]) d3 = {x: x**2 for x in range(5)} # {0:0, 1:1, 2:4, 3:9, 4:16}

字典操作

student = {"name": "张三", "age": 20} # 访问 student["name"] # "张三" student.get("phone", "N/A") # 安全访问, 默认值 # 修改 / 添加 student["age"] = 21 # 修改 student["phone"] = "123" # 新增 # 删除 del student["phone"] student.pop("age") # 遍历 for key, value in student.items(): print(key, value)

字典: 交互式练习

字典操作台

{"name": "Alice", "age": 20, "city": "Beijing"}

字典推导式

# 平方字典 squares = {x: x**2 for x in range(6)} # {0:0, 1:1, 2:4, 3:9, 4:16, 5:25} # 反转字典 original = {"a": 1, "b": 2} reversed_d = {v: k for k, v in original.items()} # {1:"a", 2:"b"}

字典 vs 列表: 何时选择?

用字典: 需要按名称(key)快速查找
用列表: 数据有序且需要按位置访问
字典查找: O(1) 平均时间复杂度
列表查找: O(n) 需要遍历

元组 (Tuple): 不可变的序列

元组的特征

有序: 元素有固定的位置和索引
不可变: 创建后不能修改、添加或删除
可重复: 允许包含重复的元素
可哈希: 可以作为字典的键(如果元素也不可变)
# 创建元组 point = (3, 4) colors = ("red", "green", "blue") single = (42,) # 注意逗号! not_tuple = (42) # 这是int, 不是tuple # 元组解包 x, y = (3, 4) a, b, c = colors # 交换 (本质是元组解包) a, b = b, a

元组操作

t = (1, 2, 3, 2, 1) # 访问 t[0] # 1 t[-1] # 1 t[1:3] # (2, 3) # 方法 (只有两个!) t.count(2) # 2 t.index(3) # 2 # 不可变意味着: # t[0] = 99 → 报错! TypeError

元组 vs 列表: 何时选择?

用元组: 数据不应被修改(坐标、RGB颜色、数据库记录)
用元组: 需要作为字典的键
元组优势: 比列表更节省内存, 速度更快

集合 (Set): 唯一性与集合运算

集合的特征

无序: 没有索引, 不能按位置访问
唯一: 自动去重, 不允许重复元素
可变: 可以添加和删除元素
元素不可变: 元素必须是不可变类型
# 创建集合 s = {1, 2, 3, 3, 2} # {1, 2, 3} empty = set() # 不能用 {}, 那是空字典 # 从列表去重 nums = [1, 2, 2, 3, 3, 3] unique = set(nums) # {1, 2, 3} # 集合推导式 s = {x**2 for x in range(5)} # {0, 1, 4, 9, 16}

集合运算

a = {1, 2, 3, 4} b = {3, 4, 5, 6} # 并集 (所有元素) a | b # {1,2,3,4,5,6} a.union(b) # 交集 (共同元素) a & b # {3,4} a.intersection(b) # 差集 (a有b没有) a - b # {1,2} a.difference(b) # 对称差集 (不共有的) a ^ b # {1,2,5,6}

类型转换与嵌套结构

类型互转

# list ↔ tuple ↔ set nums = [1, 2, 3] t = tuple(nums) # (1, 2, 3) s = set(nums) # {1, 2, 3} back = list(s) # [1, 2, 3] # dict → list d = {"a": 1, "b": 2} keys = list(d.keys()) # ["a","b"] vals = list(d.values()) # [1, 2] pairs = list(d.items()) # [("a",1),("b",2)] # list → dict d2 = dict(pairs) # {"a":1, "b":2}

嵌套数据结构

# 列表嵌套字典 (常见!) students = [ {"name": "Alice", "score": 95}, {"name": "Bob", "score": 87}, {"name": "Carol", "score": 92} ] # 访问嵌套数据 students[0]["name"] # "Alice" # 字典嵌套列表 class_info = { "names": ["Alice", "Bob"], "scores": [95, 87] }
